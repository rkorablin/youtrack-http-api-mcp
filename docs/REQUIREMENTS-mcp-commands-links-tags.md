# Требование: доработка MCP YouTrack — связи задач, теги, projectId

**Назначение:** документ для передачи агенту/разработчику на реализацию в репозитории `youtrack-http-api-mcp`.

**Источник контекста:** использование MCP из Cursor при импорте декомпозиции в проект YouTrack `IAG` (задачи создаются, связи и теги — нет).

---

## 1. Контекст и проблема

### 1.1 Создание задач (`create_issue`)

- Работает при передаче **числового id проекта** (например, `81-219`).
- Передача **shortName** проекта (`IAG`) приводит к ошибке API: `Invalid structure of entity id: IAG`.

### 1.2 Связи и теги (Commands API)

Операции **`link_issues`** и **`manage_issue_tags`** (и, вероятно, другие вызовы на базе Commands API) возвращают **HTTP 400** с сообщением вроде:

- `Property CommandList.issues is required`
- `Свойство CommandList.issues является обязательным`

Из-за этого нельзя автоматизировать типовой сценарий:

- подзадачи к эпику (**Subtask of**);
- зависимости (**Depends on** / **Is required for**);
- теги (**Tags**).

---

## 2. Цель

1. **Связи между задачами** — надёжное создание связей по идентификаторам задач (`id`, `idReadable` вида `IAG-42`) и **имени типа связи**, согласованному с конфигурацией целевого YouTrack (`Subtask`, `Depends on`, и т.д.).
2. **Теги** — добавление и удаление тегов у задачи по тем же идентификаторам.
3. **Создание задач** — по возможности принимать `projectId` как **shortName** (`IAG`) **и** как внутренний id (`81-219`), с нормализацией на стороне MCP (разрешение shortName → id через API проекта).

---

## 3. Функциональные требования

| ID | Требование |
|----|------------|
| **FR-1** | `link_issues`: запрос к YouTrack должен формировать **корректное тело Commands API**, включая обязательное поле **`issues`** (список идентификаторов задач, к которым применяется команда), если это требуется используемой версией/инстансом YouTrack. |
| **FR-2** | В **README** MCP задокументировать: точный формат команды/тела для связи (пример), имена типов связей в целевом инстансе или способ их получения через API. |
| **FR-3** | `manage_issue_tags`: аналогично FR-1 — корректная передача **`issues`** и команды добавления/удаления тега в синтаксисе Commands API. |
| **FR-4** | При ошибке **400** логировать **полное тело ответа** и **отправленный payload** (без токенов/секретов) для отладки. |
| **FR-5** | Добавить **тесты** (юнит с mock ответа YouTrack и/или интеграционные) с проверкой наличия `issues` в итоговом запросе к API, где это применимо. |
| **FR-6** | (Желательно) `create_issue`: поддержка `projectId` в виде **shortName** проекта с резолвом в entity id перед вызовом REST. |

---

## 4. Нефункциональные требования

- Указать в README целевую **версию YouTrack** (Server / Cloud) и протестированный сценарий.
- По возможности **идемпотентность**: повторное «добавить связь» не должен приводить к необрабатываемому состоянию; допустима явная ошибка «уже связано».

---

## 5. Критерии приёмки

1. Вызов `link_issues` с валидными идентификаторами задач и существующим в инстансе типом связи **создаёт связь** в UI YouTrack без 400.
2. Вызов `manage_issue_tags` **добавляет и снимает** тег у задачи без 400.
3. При реализации FR-6: `create_issue` с `projectId: "IAG"` **создаёт задачу** в проекте IAG.
4. В README есть **примеры** вызовов инструментов MCP для клиента (Cursor и т.п.).

---

## 6. Техническая гипотеза для реализации

Ошибка `CommandList.issues is required` указывает, что сервер ожидает структуру **`CommandList`** с массивом **`issues`**, а не запрос без явного списка задач.

Необходимо свериться с официальной документацией YouTrack REST API для применения команд (**Apply Command** / **commands**) в используемой версии: как передаётся список задач и текст команды (в т.ч. эквивалент строк вида `IAG-2 subtask of IAG-1` в JSON).

## Статус реализации (v0.2.2)

- **FR-1 / FR-3:** `POST /api/commands` формируется через `buildCommandsRequestBody` с обязательным **`issues`**; используется в `link_issues`, `manage_issue_tags`, `change_issue_assignee`.
- **FR-2 / примеры:** см. разделы *Commands API* и *Example MCP tool calls* в `README.md`.
- **FR-4:** при HTTP **400** в **stderr** пишутся полный ответ, path/method, заголовки без `Authorization`, тело запроса.
- **FR-5:** `npm test` — `test/youtrack-helpers.test.mjs`.
- **FR-6:** `create_issue` — `projectId`: `^\d+-\d+$` → `{ id }`, иначе → `{ shortName }` (`lib/youtrack-helpers.mjs`).
